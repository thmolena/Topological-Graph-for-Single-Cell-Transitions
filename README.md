## TL;DR

A single-cell dataset is a **graph**: each cell is connected to its nearest neighbours
on the expression manifold, and cell-state *transitions* live along that graph's topology.
This project forecasts a cell's next state by **propagating a handful of annotated labels over
the kNN cell graph**, and adds a **topology-aware active-sampling policy** that decides *which
unlabeled cells to annotate next* to recover a **rare** cell state. On a controlled synthetic
branching lineage — held out by **donor**, **batch**, **pseudotime**, and **perturbation
condition** — graph smoothing beats a non-graph kNN classifier in the scarce-label regime,
and active sampling lifts rare-state recall far above random querying.

Every number in this README is generated by the included code (`make demo`); nothing is hand-edited.

## Method

```
 cells (expression embedding) ──► kNN cell graph ──► label propagation ──► next-state forecast
        donor·batch·pert·time        (Betti-0 vs k;      (graph-smoothed;        │
        metadata + a RARE state)      sklearn kNN)        kNN baseline)           ▼
                              topology-aware active sampling  ◄──── rare-state recall
                              (uncertainty × inverse density)      (vs random querying)
```

1. **Synthetic lineage** ([`synthetic.py`](code/src/topocell/synthetic.py)) — cells progress along a tree of
   2–3 fates over pseudotime as Gaussian blobs, carrying branch/state, pseudotime, donor,
   batch and perturbation labels, plus a deliberately **rare state** (a small late-branch
   subpopulation). Fully seeded; this is the sanity benchmark. Real data plugs in through
   honest [`ingest.py`](code/src/topocell/ingest.py) stubs (scanpy / AnnData) — never fabricated.
2. **Cell graph** ([`graph.py`](code/src/topocell/graph.py)) — a symmetric kNN graph; its number of connected
   components (a Betti-0 / \(H_0\) proxy) is **monotone non-increasing in `k`**, a real
   topological invariant asserted in [`test_betti_monotonic.py`](code/tests/test_betti_monotonic.py).
3. **Forecasting + active sampling** ([`policy.py`](code/src/topocell/policy.py)) — graph-smoothed label
   propagation (the contribution) vs a plain kNN classifier (the apples-to-apples baseline);
   an active-sampling policy that scores cells by uncertainty × inverse local density to find
   the rare state, compared to random querying.
4. **Audited metrics** ([`metrics.py`](code/src/topocell/metrics.py)) — accuracy, expected calibration error
   (ECE), and rare-state recall, under four leakage-checked split protocols.

## Results (smoke scale — regenerate with `make demo`)

| split | method | accuracy | ECE | rare-state recall |
|---|---|---|---|---|
| donor | **graph-smoothed** | **0.7048** | 0.1886 | **0.7500** |
| donor | baseline kNN | 0.5048 | 0.1410 | 0.0000 |
| batch | **graph-smoothed** | **0.6961** | 0.1479 | **1.0000** |
| batch | baseline kNN | 0.3676 | 0.1039 | 0.0000 |
| time | **graph-smoothed** | **0.7514** | 0.2903 | 0.0000 |
| time | baseline kNN | 0.2260 | 0.2682 | 0.0000 |
| perturbation | **graph-smoothed** | **0.6277** | 0.1582 | **0.8333** |
| perturbation | baseline kNN | 0.5182 | 0.0808 | 0.0000 |

*400 cells, 4 states (5% rare), 10% annotated, k=15 — the laptop-scale smoke configuration
(~0.2 s).* Headline: graph-smoothed mean accuracy **0.6950** vs baseline **0.4042**
(**+0.2908**); **topology-aware active sampling** reaches rare-state recall **0.833** vs
**0.000** for random querying at the same budget. Graph propagation trades a little
calibration (higher ECE) for the large accuracy and rare-recall gains — both reported, not
hidden. The reported-scale numbers (3000 cells, scarcer labels, 5 seeds, tight CIs) come from
`make full-run` with [`configs/full.yaml`](code/configs/full.yaml); there the accuracy gain
is **+0.06** and the active-sampling rare-recall gain is **+0.33** (non-overlapping 95% CIs).
The Betti-0 curve, per-split breakdown and integrity flags are written to
`results/summary.json` and plotted in `figures/`.

### Library — `code/src/topocell/`, file by file

| File | Contribution |
|---|---|
| [`synthetic.py`](code/src/topocell/synthetic.py) | Seeded synthetic **branching lineage** — cells along a tree of fates over pseudotime, carrying branch/state · donor · batch · perturbation, plus a deliberately **rare state**. The sanity benchmark every result is built on. |
| [`ingest.py`](code/src/topocell/ingest.py) | Honest **STUB** hooks for real AnnData / perturbation-screen data (scanpy). Defines where real data plugs in — never fabricates it. |
| [`graph.py`](code/src/topocell/graph.py) | The symmetric **kNN cell graph**, its component count (Betti-0 / \(H_0\) proxy) vs `k`, and local density. **The topology the whole method rests on.** |
| [`splits.py`](code/src/topocell/splits.py) | The four held-out split protocols — **donor · batch · time · perturbation** — plus an explicit leakage check, so transfer claims are real. |
| [`policy.py`](code/src/topocell/policy.py) | Graph-smoothed **label propagation (the contribution)** vs a plain kNN baseline, and the **topology-aware active sampler** (uncertainty × inverse local density) vs random querying. |
| [`metrics.py`](code/src/topocell/metrics.py) | Accuracy, expected calibration error (ECE), rare-state recall, and the per-split summaries. |
| [`runner.py`](code/src/topocell/runner.py) | Orchestrates the full protocol end to end and writes `results/summary.json`. |
| [`audit.py`](code/src/topocell/audit.py) | The readiness gate: scans for forbidden superiority claims and verifies every reported number traces to a generated artifact. |
| [`plotting.py`](code/src/topocell/plotting.py) | Renders the lineage scatter and forecast / rare-recall figures into `figures/`. |
| [`config.py`](code/src/topocell/config.py) · [`seed.py`](code/src/topocell/seed.py) | YAML config loading and deterministic seeding with run provenance (reproducibility plumbing). |

## How to use

```bash
cd code
make install         # numpy / scipy / networkx / scikit-learn / matplotlib (CPU-only)
make test            # 15 tests incl. no-leakage splits + Betti-0 monotonicity
make demo            # ~0.2 s: writes results/summary.json
make tables figures  # results/main_results.{tex,md}, figures/*.pdf
make audit           # readiness gate: traceable numbers, integrity flags, no forbidden claims
make full-run        # reported-scale configuration
```

Or open [`index.html`](index.html) directly in any browser to explore the method interactively — it
re-implements the kNN cell graph, the normalized label-propagation diffusion, and the
topology-aware active sampler in pure JavaScript, so what you see on the page is the same math
as the Python package.

## GPU / acceleration note

The reference implementation is **CPU-only and runs everywhere**. The architecture exposes
optional accelerated backends that can be installed separately:
a **RAPIDS cuML** GPU kNN build and **cuGraph** connected-components for million-cell atlases,
a **PyTorch-Geometric** message-passing GNN as a learned drop-in for the label-propagation
forecaster, and GPU-batched graph diffusion. None are required to reproduce the results.
