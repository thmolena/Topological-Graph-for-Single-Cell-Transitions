# Spectral-Truncated Noncommutative Graph Operators for Single-Cell Transitions

A pseudotime-directed, spectrally truncated graph operator for few-label single-cell
state-transition forecasting and rare-state recovery.

[Live in-browser demo](https://thmolena.github.io/P4-Topological-Graph-RL-for-Single-Cell-Transitions/)
· [Manuscript (PDF)](submission/main.pdf) · Python 3.9–3.13 · MIT License

## Abstract

Forecasting how single cells move between states—across donors, batches, timepoints and
perturbations—is confounded by nuisance variation and by the rarity of the states that matter most.
The standard graph smoother for this task diffuses labels through the *self-adjoint* normalized
adjacency of the cell *k*-nearest-neighbour (*k*NN) graph, which is blind to the arrow of
differentiation and, when evaluated transductively, conflates diffusion with seeing the held-out
cells' connectivity. This work introduces a different object—a **spectral-truncated,
pseudotime-directed graph operator**. Edges are reweighted by a bounded forward kernel of the
pseudotime increment and row-normalized, producing a **non-self-adjoint (noncommutative) propagator**
that transports labels along the trajectory; the diffusion is then restricted to the leading *r*
graph-Fourier modes—a **spectral truncation** that band-limits and denoises the operator—and, through
an out-of-sample Nyström extension, is made **inductive**, so held-out connectivity is never used at
inference. The construction is the single-cell instance of the operator noncommutativity and spectral
truncation studied in C\*-algebraic kernel machines, carried into a single-cell transition benchmark
and equipped with a distribution-free conformal layer and a genuine persistent-homology topology
signal. The benchmark is a fully seeded synthetic branching lineage carrying donor, batch, pseudotime
and perturbation metadata; transfer is evaluated under four leakage-controlled held-out split
protocols. Every reported figure, table and number is generated by the accompanying pip-installable
package and traced by an audit gate; none is hand-edited.

## Most novel contribution

> **A pseudotime-directed, spectrally truncated, inductive graph operator that recovers rare cell
> states and rescues forward-in-time extrapolation, while resolving the transductive confound of
> symmetric label propagation.**

The operator differs from the standard single-cell smoother in three coupled ways, each tied to a
measured effect. (i) **Directionality (noncommutativity).** Reweighting each edge by a bounded forward
kernel of the pseudotime increment and row-normalizing yields a non-self-adjoint propagator
`P = D_W⁻¹W` whose relative non-normality `‖PPᵀ − PᵀP‖_F / ‖P‖_F²` is strictly positive and increases
with the directional strength; this directed transport is what carries labels along the trajectory.
(ii) **Spectral truncation.** Restricting the diffusion to the leading *r* graph-Fourier modes is the
provably optimal rank-*r* low-pass approximation of the symmetric backbone, and the truncated
multiplication operators on the cell graph do not commute—the spectral-truncation noncommutativity of
C\*-algebraic kernel machines, instantiated on single-cell data. (iii) **Inductive extension.** An
out-of-sample Nyström extension labels held-out cells from the training field alone, so the
comparison is inductive on both sides and the accuracy gain is not the transductive information
advantage that inflates symmetric-smoothing studies—it is the matched inductive control whose absence
those studies identify as their decisive gap.

## Main results

Values are transcribed from `submission/code/results/summary.json` at the **reported scale**
(`submission/code/configs/full.yaml`: 3000 cells, 4 states, rare-state fraction 0.04, `k = 20`,
truncation rank `r = 80`, label fraction 0.03, 5 seeds; mean wall-clock runtime 32.0 s and peak
memory 621.2 MB on the recorded platform).

| split | inductive STDD (accuracy) | *k*NN baseline (accuracy) | inductive STDD (rare recall) | *k*NN (rare recall) |
|---|---|---|---|---|
| donor | **0.7227** | 0.6550 | **0.3968** | 0.0000 |
| batch | **0.7134** | 0.6355 | **0.5706** | 0.0320 |
| time | **0.6323** | 0.4574 | 0.0000 | 0.0000 |
| perturbation | **0.6741** | 0.6490 | **0.3616** | 0.0286 |

- The inductive operator improves held-out forecast accuracy over the matched inductive *k*NN
  classifier on **all four** transfer protocols (mean **0.6856** vs **0.5992**, a gain of **0.0864**),
  and recovers the rare state far more often (mean rare-state recall **0.3322** vs **0.0151**).
- Directional transport is decisive for forward-in-time extrapolation: time-split accuracy rises by
  **0.1749** over the point classifier and by **0.0849** over symmetric label propagation.
- A split-conformal layer attains realized marginal coverage **0.8968** against a target of **0.90**
  on an exchangeable hold-out, and temperature scaling reduces expected calibration error from
  **0.1375** to **0.0369**.
- A persistence-guided active sampler raises rare-state recall to **0.3941** versus **0.0481** for
  random querying at the same labelling budget.

All numbers above, the per-split confidence intervals, the operator non-normality and
truncation-commutator diagnostics, the degree-0 persistence barcode and the integrity flags are
written to `submission/code/results/summary.json` and rendered in
`submission/code/figures/`; the manuscript `submission/main.tex` consumes them directly.

## Demo

The manuscript is the worked demonstration of the method. Build it from the regenerated artifacts:

```bash
cd submission/code
pip install .                                   # installs the topocell package
python scripts/run.py --config configs/full.yaml --out results   # writes summary.json
python scripts/make_tables.py && python scripts/make_figures.py  # tables + figures
cd .. && tectonic main.tex                       # or: latexmk -pdf main.tex
```

A dependency-free [in-browser demo](index.html) reproduces the headline comparison interactively.

## Reproduction

The pipeline regenerates the results artifact, tables and figures from a deterministic seeded run
(two runs produce a byte-identical `summary.json` modulo provenance):

```bash
cd submission/code
python scripts/run.py --config configs/full.yaml --out results   # reported scale
python scripts/run.py --config configs/smoke.yaml --out results   # laptop-scale smoke run
python scripts/make_tables.py     # results/main_results.tex, results/extended_results.tex
python scripts/make_figures.py    # figures/fig_{schematic,lineage,forecast,operator,calibration}.pdf
python scripts/audit_claims.py    # readiness gate: traceable numbers, integrity flags
pytest tests/                     # property tests for the operator, conformal and persistence layers
```

## Library modules — `submission/code/src/topocell/`

| Module | Role |
|---|---|
| `synthetic.py` | Seeded synthetic branching lineage with donor, batch, perturbation, pseudotime and a rare state. |
| `graph.py` | Symmetric *k*NN cell graph and per-cell density. |
| `operators.py` | The method: graph-Fourier basis, pseudotime-directed propagator, spectral truncation, transductive and inductive spectral-truncated directed-diffusion forecasters, non-normality and truncation-commutator diagnostics. |
| `conformal.py` | Split-conformal prediction sets and temperature scaling for distribution-free rare-state coverage. |
| `persistence.py` | Exact degree-0 persistence barcode (union-find / MST) and persistence-based topology features. |
| `policy.py` | Symmetric label-propagation and *k*NN baselines; persistence- and density-aware active samplers. |
| `splits.py` | Four leakage-checked transfer protocols (donor, batch, time, perturbation). |
| `metrics.py` | Accuracy, expected calibration error, rare-state recall and per-split summaries. |
| `runner.py` | End-to-end driver; writes `results/summary.json`. |
| `plotting.py` | The five manuscript figures. |
| `ingest.py` | Honest stub hooks for real AnnData / perturbation-screen ingestion. |
| `config.py`, `seed.py`, `audit.py` | Configuration, deterministic seeding with provenance, and the claim-audit gate. |
